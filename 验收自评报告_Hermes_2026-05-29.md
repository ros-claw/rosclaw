## 四、已知缺口（诚实填写）

### 🔴 P0 阻塞（必须修复）

1. **How 模块几乎为空** — `how/rules.py` 只有骨架，`RecoveryHint`/`参数 patch`/`重试对比` 均未实现。影响 L5 和场景 B/C 的失败恢复。
2. **Practice 无 episode 记录** — 无 `rosclaw practice list/show/replay` CLI 命令，无 artifact 目录生成逻辑。
3. **无 install.sh 一键安装** — 用户无法从零启动，PEP668 问题未处理。
4. **Claude Code MCP 闭环未验证** — 所有 demo 都是手动/测试驱动，非真实 MCP 工具调用闭环。
5. **Sandbox 防火墙拦截未验证** — `ActionGuard`/`SchemaGuard` 有代码但无真实危险动作拦截验证。

### 🟡 P1 重要（强烈建议修复）

6. **VLM/Critic Provider 缺失** — 场景 C（抓取）和 D（巡检）需要视觉感知和成功判断能力。
7. **Know 模块未接入主流程** — `know/graph.py`/`know/storage.py` 有实现但 Runtime 主流程未调用。
8. **Dashboard 未真实启动验证** — 有 `DashboardServer` 代码但未在真实 runtime 场景中运行。
9. **Swarm 未实战** — 多 agent 协调代码存在但未在真实多机器人场景中验证。
10. **G1 free-floating 行走** — 当前为固定底座 demo。

### 🟢 P2 改进（发布后可补）

11. 文档完整性（API 文档、教程）
12. 更多机器人 profile（H1、B2-W 等）
13. Docker 打包
14. CI/CD 流水线
