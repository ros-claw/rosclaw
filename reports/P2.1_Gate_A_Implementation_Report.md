### 1.2 ROSClaw-rich safety metadata (Gate A)

- **NormalizedPracticeEpisode v2** — schema bumped to `rosclaw.practice.normalized.v2` with `safety`, `failure`, `intervention`, `action_context`, `done`, `success` per-frame blocks and robot/task metadata.
- **Extension schema & vocab** — `meta/rosclaw/schema.json` and `meta/rosclaw/vocab.json` for categorical encoding (`rosclaw.sandbox.decision`, `rosclaw.failure.code`, `rosclaw.intervention.source`, `rosclaw.action.source`).
- **Feature inference** — `dataset_feature_infer.py` emits ROSClaw features only when the profile requests them and the episode provides data.
- **Profiles** — `minimal` (default), `safety`, `physical`, `safety-rich`; optional `--include <groups>` override.
- **Episode sidecar** — `meta/rosclaw/episodes.parquet` with success, failure code, intervention counts, sandbox block frames, etc.
- **Body snapshot** — `--include-body-snapshot --body-snapshot-mode {none,sanitized,full}`; sanitized mode strips serial numbers, IP addresses, tokens, geo coordinates, and home paths.
- **Dry-run** — `--dry-run` previews features/warnings without writing.
- **Rich validation** — `--level {structural,load,dataloader,rich}`.

### 1.3 Gate A.1 cleanup

- **Profile completeness enforcement**
  - `dataset_profile.py` distinguishes `AVAILABLE_PROFILE_NAMES` from `PLANNED_PROFILE_NAMES` (`physical` is planned/unavailable).
  - `resolve_profile(..., allow_unavailable=False)` rejects unavailable profiles with a clear error.
  - `run_dataset_worker_op()` enforces availability before building the worker request.
  - `_enforce_profile_completeness()` compares requested vs. written feature groups.
  - Exports fail by default when groups are missing; `--allow-partial` permits partial exports with a warning.
  - Worker response and export report include `requested_feature_groups`, `written_feature_groups`, `missing_feature_groups`, and `profile_satisfied`.
  - Export report `profile` block contains `name`, `requested`, `validated`, `missing`, `planned`, `satisfied`, and `scope`.

- **Missingness semantics**
  - Boolean safety/failure/intervention/action/outcome fields are encoded as `int8` with values:
    - `-1` = UNKNOWN / not recorded
    - `0`  = FALSE / INACTIVE
    - `1`  = TRUE / ACTIVE
  - `risk_score` uses `float32` with `NaN` for unknown.
  - Categorical enums keep the `0 = UNKNOWN` vocabulary convention.
  - `NormalizedSafety.modified`, `NormalizedFailure.active`, `NormalizedIntervention.active`, `NormalizedActionContext.was_clamped`, `NormalizedFrame.done`, and `NormalizedFrame.success` are now `bool | None = None` so unknown is explicit.

- **Body snapshot provenance & security**
  - `body_snapshot.py` rewritten with SHA-256 source/sanitized hashes.
  - Manifest records redaction count and redacted fields.
  - Full mode requires `--acknowledge-sensitive-body-data`; without it a `SensitiveBodyDataError` is raised.
  - CLI passes the acknowledgement flag through from both `practice export` and `lerobot export-dataset`.

- **Validation scope in doctor/report**
  - `dataset_report.py` adds `IMPLEMENTED_FEATURE_GROUPS` and `_profile_scope()`.
  - `DatasetExportReport.from_dict()` now restores `profile` and `extension_gate` (fixed empty-profile bug).
  - `DatasetWorkerResponse` carries the `profile` name end-to-end so reports display the correct profile.
  - `get_dataset_export_validation_status()` returns `profile` and `extension_gate` in both `validated` and `stale` states.
  - `rosclaw lerobot doctor` prints profile name, requested/validated/missing groups, and extension gate.

- **Events sidecar**
  - New `dataset_events.py` writes `meta/rosclaw/events.parquet` (or JSONL fallback).
  - One row per event per frame for sandbox decisions, failure/intervention activation, action provenance, and episode outcome.
  - Events are materialized from normalized episode data so downstream tools do not need to decode integer columns.

## 6. Issues found and fixed during implementation

| Issue | Root cause | Fix |
|-------|------------|-----|
| Real worker export failed with `image_file_not_found` when invoked through the runner | `LeRobotDatasetWorkerRunner._copy_normalized_bundle()` used `metadata.source_dir` even when it was an empty string, resolving image paths against the current working directory instead of the episode JSON's parent directory. | Updated `_copy_normalized_bundle()` to default to `src_json.parent` and only override with `source_dir` when it is a non-empty, existing path. |
| Pre-existing doctor tests failed after adding real-runtime fixture | The session-scoped autouse fixture registered the real LeRobot runtime for every test, breaking tests that expected no runtime. | Converted the fixture to non-autouse (`real_lerobot_runtime_config`) and added `@pytest.mark.usefixtures("real_lerobot_runtime_config")` only to tests that need the real runtime. |
| Dry-run exported a real dataset | The dry-run body in `cmd_lerobot_export_dataset` was indented inside the `except ValueError` block after `return 1`, so it fell through to the real export path. | Moved dry-run inference into the main `try` block before the real export call. |
| Real worker export failed with `NameError: _encode_label` | `_encode_label` was accidentally removed when `_encode_bool` was added. | Restored `_encode_label` before `_encode_bool`. |
| Normalizer tests expected `False` defaults | Defaults changed to `None` to represent unknown. | Updated tests to assert `None` and added an explicit unknown-done/success case. |
| Doctor showed empty `"profile": {}` | `DatasetExportReport.from_dict()` did not restore the `profile` or `extension_gate` fields. | Updated `from_dict()` to pass `profile` and `extension_gate`. |
| Report profile name always showed `minimal` | `DatasetWorkerResponse` did not carry the worker's `profile` field, so `report_from_worker_response` fell back to `minimal`. | Added `profile: str` to `DatasetWorkerResponse` and included it in `to_dict()`/`from_dict()`. |
