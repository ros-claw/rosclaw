# ROSClaw × LeRobot Bridge — P2.1 Gate B Implementation Report

**Date:** 2026-07-11
**Scope:** P2.1 Gate B — Physical telemetry, units/feature-names sidecars, time sync, and missing-data policy.
**Status:** Implementation complete, all integration tests passing, manual smoke commands passing.

## 1. What was delivered

### 1.1 Physical telemetry fields

- **NormalizedPracticeEpisode v2 extended** with optional per-frame telemetry arrays:
  - `observation.motor_current` (float, per motor)
  - `observation.joint_temperature` (float, per joint)
  - `observation.force_torque` (float wrench `[Fx, Fy, Fz, Tx, Ty, Tz]`)
  - `observation.contact` (bool/int8 per contact sensor)
  - `observation.joint_velocity` (float, per joint)
  - `observation.joint_effort` (float, per joint)
- **Dimension validation** in `_validate_frames()` ensures each telemetry array has a consistent length across frames.
- **Feature inference** in `dataset_feature_infer.py` dynamically shapes telemetry features from the first non-empty frame.
- **Worker encoding** in `dataset_worker_main.py` maps telemetry arrays into LeRobot float32/int8 features and fills missing values according to `--missing-policy`.

### 1.2 `physical` profile

- `dataset_profile.py` now marks `physical` as **available**.
- `physical` includes feature groups `safety`, `action`, and `physical_telemetry`.
- `dataset_worker_main.py` profile mapping updated accordingly.

### 1.3 Units and feature-names sidecars

- New `dataset_units.py` with default SI-ish units and human-readable axis names.
- Writes:
  - `meta/rosclaw/units.json`
  - `meta/rosclaw/feature_names.json`
- Integrated into `LeRobotDatasetWorkerRunner._write_sidecars()` using the actual exported feature keys.

### 1.4 Time sync and `sync_stats.parquet`

- `NormalizedFrame` now carries optional time-sync fields:
  - `source_timestamp_ns`
  - `clock_domain`
  - `episode_time_sec`
- New `dataset_sync.py` builds per-episode timing statistics:
  - start/end timestamps and source timestamps
  - min/max/mean source deltas
  - missing timestamp frame counts
  - clock-domain consistency
- Integrated into `_write_sidecars()` as `meta/rosclaw/sync_stats.parquet` (or JSONL fallback).

### 1.5 Missing-data policy

- New CLI flag `--missing-policy {error,drop-frame,fill-last,nan}` on both `rosclaw practice export` and `rosclaw lerobot export-dataset`.
- Default is `nan`.
- `error` immediately fails the export when a telemetry field is absent for a frame that should contain it.
- The policy is propagated through `run_dataset_export()` -> `run_dataset_worker_op()` -> `DatasetWorkerRequest.features` -> `dataset_worker_main.py`.
- Runner now preserves structured worker error responses even when the worker exits with a non-zero code.

### 1.6 Tests and fixtures

- New example fixture: `examples/practice/physical_lerobot_episode/episode.json` with full telemetry arrays.
- New integration test: `tests/integrations/test_lerobot_physical_telemetry_export.py`
  - Validates telemetry features, dimensions, sidecars, DataLoader smoke, and `error` missing-data policy.
- Extended `tests/integrations/test_lerobot_practice_normalizer_v2.py` with telemetry round-trip and dimension-mismatch tests.
- Extended `tests/integrations/test_lerobot_dataset_feature_infer.py` with physical telemetry inference tests.

- `dataset_worker_main.py` remains standalone with no `rosclaw` imports.
