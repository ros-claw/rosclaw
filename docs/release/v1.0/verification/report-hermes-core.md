## 4. 建议修复措施

### ISSUE-001 (Medium)
- 在 `README.md` 和安装文档中明确说明需要虚拟环境
- 提供 `python3 -m venv .venv && source .venv/bin/activate && pip install -e .` 作为推荐安装方式
- 或在 `pyproject.toml` 中配置 `pip` 安装提示

### ISSUE-002 (Low)
- 实现 `rosclaw status` 子命令，返回 Runtime 当前状态、已加载模块、内存后端等关键信息
- 参考 `Runtime.state` 和各模块 `is_running` 属性实现

### ISSUE-003 (Low)
- 在 Sandbox fallback 到 mock 模式时添加 **警告日志** (`logger.warning`)
- 明确告知用户当前未使用真实物理引擎，仅运行 mock 仿真

### ISSUE-004 (Info)
- 更新官方文档和测试脚本，明确 `Runtime` 生命周期方法为同步 API
- 避免后续用户/测试因 async 假设而产生混淆
