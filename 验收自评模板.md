## 三、已知缺口（必须诚实填写）

1. **端到端物理任务闭环未跑通**：所有模块单元测试通过，但缺少"一个真实物理任务从 Agent → Provider → Sandbox → Runtime → Practice → Memory"的完整运行证据。当前只有单元/集成测试，没有场景级验收测试。

2. **Sandbox 仿真执行后端不完整**：MuJoCo 引擎初始化代码存在，但 `SandboxRuntimeAdapter` 在真实 e-URDF 加载时报错 (`Robot 'ur5e' not found`)，说明 sandbox-world-robot 的注册链路有缺口。

3. **Provider 路由器缺少真实 inference 适配器**：ProviderRegistry 能注册 mock provider，但 LLM/VLM/VLA 的真实推理适配器（OpenAI、Qwen、本地模型等）未接入，所有 provider 调用目前只能返回 mock 结果。

4. **Dashboard 不存在**：没有 Web UI 或实时观测页面。`rosclaw dashboard` CLI 命令未实现。

5. **Practice 缺少 artifact 生成**：虽然 artifacts 表存在，但 MCAP 录制、视频生成、replay 文件实际落盘逻辑未在真实 episode 中验证。

6. **How 缺少自动触发机制**：HeuristicEngine 能查询规则，但 failure event → how recovery → parameter patch → retry 的自动化链路未建。

7. **Event Bus 缺少持久化和外部接入**：当前 EventBus 是内存型，不支持跨进程/跨机器的事件流，也没有 NATS JetStream 等真实消息中间件适配。

## 四、预计修复工时

- **端到端闭环任务（A/B/C 三选一跑通）**：40–60 小时
  - Sandbox MuJoCo 后端修复：8h
  - Provider 真实推理适配器（至少 1 个 LLM + 1 个 VLM）：16h
  - Reach/Grasp/PID skill 实际执行逻辑：16h
  - Practice episode 完整记录 + artifact 生成：8h
  - Dashboard 最小可用版本：12h
- **How 自动恢复闭环**：16–24 小时
- **Event Bus 持久化**：8–12 小时
- **6 个场景全部验收通过**：约 120–160 小时（2–3 人周）

**总计：约 160–200 工时（1 人月）才能达到 v1.0 RC 标准**

## 五、需要其他模块配合的事项

1. **rosclaw-sandbox**：需要修复 `SandboxRuntimeAdapter` 的 e-URDF 加载问题，并提供至少一个可运行的 MuJoCo world 模板。

2. **rosclaw-provider**：需要提供至少一个真实 LLM/VLM provider 适配器（OpenAI/Qwen/local），而不是仅 mock。

3. **rosclaw-runtime**：需要验证 `rosclaw start/stop/doctor/status` 在干净环境中的可用性，并修复 `test_provider_loader_custom_class_valid` 等阻塞测试。

4. **rosclaw-dashboard**：需要新建。当前完全缺失。

5. **rosclaw-practice**：需要完善 episode 的 artifact 落盘逻辑（MCAP/jsonl/video）。
